# TASK 4 Documentation

## Objective
Build a version-controlled DevOps project with Git best practices.

## Branch Structure
- main
- dev
- feature

## Workflow
1. Work in feature branch
2. Merge feature branch to dev via pull request
3. Test changes
4. Merge dev into main
5. Tag releases

## Deliverables
- README.md
- .gitignore
- Branches
- Pull Requests
- Tags
- Documentation
